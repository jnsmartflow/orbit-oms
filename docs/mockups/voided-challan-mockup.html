Spec notes:

  • Watermark: Diagonal -22°, font-weight 900, large red text with light red fill rgba(220,38,38,0.16) and a red border box. pointer-events: none so it doesn't intercept clicks. Lives inside .doc with overflow: hidden so it stays clipped to the document.
  • Watermark hidden in print. Per spec ("Print & PDF disabled") the print path is blocked at the button. But also wrap the watermark element with @media print { .void-mark { display: none } } as a belt-and-braces measure in case anyone screenshots or saves PDF via browser.
  • Print button: disabled, gray bg, tooltip "Cannot print a voided challan". The handler that calls window.print() must also short-circuit if isVoided === true.
  • Edit button: same disabled treatment. Server-side, the formula POST handler (Phase 2c, ~line 395) already 409s on voided challans — UI mirrors that.
  • Red banner: Shows reason (human label, not enum code), remark, who-and-when, plus a note that the linked OBD was removed. Banner uses bg-red-50 / border-red-200 / text-red-900. Lives between action bar and document.
  • Left panel voided row: faded (0.65 opacity), OBD strikethrough, "Voided" pill in red, challan number badge tinted red too. Still clickable so user can inspect.
  • Voided rows in panel: stay in the list. Not hidden. Chandresh needs to see what got voided when reviewing a day.
