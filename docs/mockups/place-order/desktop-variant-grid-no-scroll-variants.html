Variant Grid — No-Scroll Design Options

Two layout variants for fitting all bases on a single laptop viewport (1440×900 reference). Variant A collapses the speed dial when a sub-product is active. Variant B keeps the speed dial and compresses cells instead. Both use pagination (●●●) for sub-products with >13 bases — only GLOSS triggers it.

Comparison · pros / cons
Variant A — Collapsed speed dial
  • ✔ Cells stay readable (56×36px, font 13.5px)
  • ✔ ~14-16 bases fit at full cell size
  • ✔ Speed dial as breadcrumb keeps context
  • ✖ Mode switch: full grid ↔ breadcrumb (extra state)
  • ✖ Less product browse-ability when sub-product is open
Variant B — Full speed dial, compact cells
  • ✔ Speed dial always visible (no mode switch)
  • ✔ Operator can browse families while qty-entering
  • ✖ Cells smaller (52×28px, font 12.5px) — harder to read
  • ✖ Mouse-click targets smaller (28px tall is the floor)
  • ✖ Less padding = denser feel
Pagination (both variants, for GLOSS only)
  • ● Page dots in card header show position (Page 1 of 3)
  • ● PgDn / PgUp flip pages (currently bound to sub-product nav in v4 — would need re-bind in two-page contexts, or use Shift+PgDn/PgUp)
  • ● 13 bases per page · popular-first sort puts running items on Page 1
  • ● Cart never moves, footer cue tells operator which page they're on
