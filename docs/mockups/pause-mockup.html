Operator Pause / Resume — feature mockup (REV 2)

7 surfaces · CURRENT is single-slot · TM dictates priority · UP NEXT is read-only

Locked decision: TM decides priority. The operator works the sequence top-to-bottom. The CURRENT slot always holds exactly one card (or is empty). Paused jobs sit on an amber shelf and Resume is gated — only available when CURRENT is empty. UP NEXT is read-only preview, no buttons.
